## Research Results: Running Parakeet Locally on macOS in Python

### Recommended: `parakeet-mlx` (Apple Silicon optimized)

```bash
uv add parakeet-mlx
```

```python
from parakeet_mlx import from_pretrained
model = from_pretrained("mlx-community/parakeet-tdt-0.6b-v3")
result = model.transcribe("audio.wav")
print(result.text)
```

- Uses Apple MLX framework with Metal/MPS acceleration
- ~190x real-time factor on M4 Pro, ~2 GB memory
- Supports streaming transcription
- Deps: `mlx>=0.22.1`, `librosa`, `numpy`, `huggingface-hub`, `ffmpeg`

### Model: `parakeet-tdt-0.6b-v3`
- 600M params, ~6.05% WER
- 25 European languages, **built-in punctuation**
- MLX variant: `mlx-community/parakeet-tdt-0.6b-v3`

### Other Options (lower priority)
| Framework | Backend | RTF | Notes |
|-----------|---------|-----|-------|
| **parakeet-mlx** ⭐ | MLX/Metal | ~190x | Best on Apple Silicon |
| onnx-asr | ONNX/CPU | ~20-50x | Lightweight, cross-platform, INT8 available |
| sherpa-onnx | C++/CPU | ~30-60x | C++ backend, ARM64 support |
| transformers | PyTorch/MPS | ~15-30x | CTC models only (no punctuation) |
| NeMo | PyTorch/CPU | ~10-20x | Full features but slowest on macOS |

### Key Takeaways
- **parakeet-mlx** is the clear winner for Apple Silicon macOS
- TDT models are the only ones with built-in punctuation (critical for dictation)
- v3 adds multilingual support over v2 (English-only)
