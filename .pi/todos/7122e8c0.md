## Implementation Plan: Dictation Pipeline

### Architecture Overview

```
┌──────────────┐     ┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
│  Microphone  │────▶│  Silero VAD  │────▶│  Parakeet MLX   │────▶│  CGEvent     │
│  sounddevice │     │  (filter     │     │  Streaming ASR  │     │  Text Inject │
│  callback    │     │   silence)   │     │  (transcribe)   │     │  (to editor) │
└──────────────┘     └──────────────┘     └─────────────────┘     └──────────────┘
    16kHz mono         VADIterator         finalized_tokens        chunked Unicode
    int16 → float32    speech segments     + draft_tokens          keystrokes
```

### Threading Model

```
Audio Thread (sounddevice callback)
  └─▶ captures mic chunks (512 samples / 32ms)
  └─▶ pushes to queue.Queue

Main Thread (event loop)
  └─▶ reads from queue
  └─▶ runs Silero VAD (< 1ms per chunk)
  └─▶ during speech: feeds chunks to StreamingParakeet.add_audio()
  └─▶ on speech end: flush, get finalized result
  └─▶ emits finalized text via CGEvent keyboard injection
  └─▶ prints draft text to terminal (optional live preview)
```

### Technology Choices

| Component | Choice | Rationale |
|-----------|--------|-----------|
| **Audio capture** | `sounddevice` | Simple callback API, PortAudio backend, numpy integration |
| **VAD** | `silero-vad` | Best accuracy, VADIterator for streaming, < 1ms latency |
| **ASR model** | `parakeet-mlx` | Native Apple Silicon (MLX/Metal), ~190x RTF, built-in punctuation |
| **Model variant** | `parakeet-tdt-0.6b-v3` | TDT = punctuation, v3 = multilingual, 600M params |
| **Text injection** | `CGEventKeyboardSetUnicodeString` (Quartz/PyObjC) | Works in ~95% of apps, full Unicode, no clipboard clobber |
| **Fallback injection** | Clipboard + Cmd+V | For apps that resist key injection |

### Implementation Steps

#### Step 1: Project setup
- `uv init` / `uv add` dependencies
- Verify mic permissions, Accessibility permissions

#### Step 2: Audio capture module (`audio.py`)
- `sounddevice.InputStream` at 16kHz mono int16, blocksize=512
- Callback pushes chunks to `queue.Queue`
- int16 → float32 conversion utility

#### Step 3: VAD module (`vad.py`)
- Load Silero VAD model
- `VADIterator` wrapper with threshold=0.5
- Returns speech start/end events
- Buffers audio during speech for ASR

#### Step 4: ASR module (`transcribe.py`)
- Load `parakeet-mlx` model (one-time, ~2GB memory)
- `model.transcribe_stream()` context manager
- Feed speech audio chunks via `add_audio()`
- Return `finalized_tokens` text on speech end
- Option: emit draft text for live preview

#### Step 5: Text injection module (`inject.py`)
- `CGEventKeyboardSetUnicodeString` with 20-char chunking
- `CGEventPost(kCGHIDEventTap, event)` for broad compatibility
- Per-character fallback mode for problematic apps
- Clipboard + Cmd+V as last-resort fallback

#### Step 6: Main orchestrator (`main.py`)
- CLI entry point with argparse
- Load model on startup (show progress)
- Main loop: queue → VAD → ASR → inject
- Ctrl+C to stop gracefully
- Optional: global hotkey to toggle on/off (e.g., hold Fn or a custom key)

#### Step 7: Testing & validation
- Test with sample audio file first (no mic)
- Test mic capture → WAV save → verify quality
- Test VAD → confirm speech boundaries
- Test streaming ASR → verify finalized vs draft
- Test text injection → type into TextEdit, VS Code, terminal
- End-to-end: speak → see text appear in editor

### Key Design Decisions

1. **Streaming ASR with VAD gating**: Only feed audio to Parakeet during detected speech. This saves compute and avoids garbage transcription from background noise.

2. **Finalized-only injection**: Only inject `finalized_tokens` text into the editor (not draft). Draft text is too volatile and would cause messy insertions. Show draft in terminal for feedback.

3. **CGEvent over clipboard**: Clipboard paste is faster but clobbers user clipboard. CGEvent keystroke injection preserves clipboard and feels more natural (text appears character-by-character).

4. **Single-process**: Keep it simple — one process, sounddevice callback thread + main thread. No need for multiprocessing given parakeet-mlx runs on Metal GPU asynchronously.

### Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Accessibility permission denied | Clear error message + instructions on first run |
| Model download slow (~600MB) | Show download progress, cache in HF hub |
| High memory (~2GB) | Acceptable for desktop; smaller model option if needed |
| Text injection fails in some apps | Clipboard fallback mode (`--clipboard` flag) |
| VAD false positives (typing noise) | Tune threshold, add min speech duration |
| Latency too high | Profile; VAD < 1ms, ASR ~5ms/chunk on M-series = acceptable |
